# What you're looking at

This screen shows everything Vex has learned — and everything it considered learning but didn't. Think of memory as a notebook of lessons whose entries fade unless real trades prove them right. The screen is read-only by design: curation is automated, and nothing here can be hand-edited.

**How memories form.** Vex cannot simply decide to remember something. Mid-conversation it can only suggest a lesson, and every suggestion is scrubbed at the door: seed phrases, private keys, and API keys cause outright rejection of the whole entry; wallet addresses are masked; live numbers like balances and prices are refused, because Vex records lessons, not snapshots. What survives becomes a pending candidate (visible in Memory candidates below). A background reviewer then applies strict rules first, with no AI involved: exact duplicates are folded into the existing memory instead of stored twice, trivial or one-off observations are kept but not promoted, and anything unsupported is held back. Only candidates that clear every rule reach an AI judge, which scores five things — grounding, durability, novelty, generality, and whether the lesson is about the decision process rather than a lucky outcome. Only a clear "promote" verdict creates a long-term memory; a broken or ambiguous judge response never does. Every verdict is logged in Manager decisions.

**How memories age.** Nothing is ever deleted. A new lesson starts on probation and stays out of Vex's automatic working context until it is confirmed a second time. From there its influence fades over roughly a month unless reinforced; a faded lesson drops out of active use but stays in the database, searchable, and returns the moment fresh evidence confirms it again. Lessons "in season" for the current market conditions fade slower; out-of-season ones fade faster.

**How trades keep memory honest.** When a trade tied to a lesson actually closes, the recorded outcome is re-checked against reality. A lesson logged as a win that closed at a loss is suppressed or flipped — automatically, in the background. Memory reflects what the ledger says happened, not what looked true at the time. Past outcomes, win or loss, are never a promise about future trades.

**The advisory-only guarantee.** Memory can shape what Vex thinks about, searches for, and mentions. It structurally cannot size a position, approve a transaction, or sign anything — the database schema simply has no value that would let a memory steer execution, and approvals stay gated by the separate approval system no matter what is stored here. A wrong lesson can misinform; it cannot move money on its own.

One habit still matters: never paste seed phrases or private keys into chat. Redaction is a safety net, not a reason to test it.
